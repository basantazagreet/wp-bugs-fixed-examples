Bug Title / Short Description (one liner) :  "Contact Form 7 form submissions just vanish — no email, no record"
Bug category: Plugin Compatibility & Integration

Description: 
A law firm client reported that their Contact Form 7 inquiry form had been silently failing for weeks. Potential clients were submitting the form, getting a success message on screen, but the firm never received any emails and there was no record of submissions anywhere.


Main Cause:
Two compounding issues were found:

A Flamingo (CF7's companion plugin) database table had corrupted entries, causing the wpcf7_submit action to throw a PHP warning that was silently swallowed — but it prevented the mail hook from firing.
A security plugin (iThemes Security) had added a rule that was blocking outbound SMTP connections on port 587, so even when emails were triggered, they were silently dropped at the server level.


What we Did:

Step 1 — Diagnosed with CF7 mail debug logging:

// functions.php — Temporarily log CF7 mail events for diagnosis
add_action('wpcf7_mail_sent', function($cf7) {
    error_log('CF7 Mail SENT: Form ID ' . $cf7->id());
});
add_action('wpcf7_mail_failed', function($cf7) {
    error_log('CF7 Mail FAILED: Form ID ' . $cf7->id());
    error_log('CF7 Mail props: ' . print_r($cf7->get_current_mail_tag(), true));
});


Log confirmed wpcf7_mail_failed was firing on every submission.



Step 2 — Repaired the Flamingo tables:

 Run via phpMyAdmin or WP-CLI (wp db query)

REPAIR TABLE wp_flamingo_inbound_messages;
REPAIR TABLE wp_flamingo_contacts;
OPTIMIZE TABLE wp_flamingo_inbound_messages;



Step 3 — Bypassed the security plugin's SMTP block by switching to a dedicated transactional email service (Mailgun) with an explicit API send method — avoiding SMTP entirely:


// functions.php — Route all WordPress mail through Mailgun API
add_action('phpmailer_init', function(PHPMailer\PHPMailer\PHPMailer $phpmailer) {
    $phpmailer->isSMTP();
    $phpmailer->Host       = 'smtp.mailgun.org';
    $phpmailer->SMTPAuth   = true;
    $phpmailer->Port       = 465;          // SSL instead of blocked 587
    $phpmailer->SMTPSecure = PHPMailer\PHPMailer\PHPMailer::ENCRYPTION_SMTPS;
    $phpmailer->Username   = 'postmaster@mg.lawfirm.com';
    $phpmailer->Password   = 'mailgun-api-key-here';
});


Step 4 — Added a submission backup to the database so there is always a record even if email fails:

// functions.php — Save every CF7 submission to a custom table as failsafe
add_action('wpcf7_before_send_mail', function($cf7) {
    global $wpdb;
    $submission = WPCF7_Submission::get_instance();
    if (!$submission) return;

    $data = $submission->get_posted_data();
    $wpdb->insert(
        $wpdb->prefix . 'cf7_backup_submissions',
        [
            'form_id'    => $cf7->id(),
            'form_data'  => json_encode($data),
            'submitted'  => current_time('mysql'),
            'user_ip'    => $submission->get_meta('remote_ip'),
        ],
        ['%d', '%s', '%s', '%s']
    );
});



All subsequent submissions were received by email and stored in the database. The firm recovered some of the lost leads by checking server logs for form POST requests.
